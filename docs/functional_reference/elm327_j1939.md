# ELM327 J1939 Heavy-Duty Protocol Reference

SAE J1939 support for heavy-duty vehicles (trucks, buses, agricultural/construction equipment).

## Overview

- **Bus**: CAN, 250 kbaud (default), 500 kbaud possible
- **ID**: Always 29-bit
- **Addressing**: Source address in ID, destination or broadcast
- **Data**: Organized by Parameter Group Numbers (PGNs) containing Suspect Parameter Numbers (SPNs)
- **ELM327 Protocol**: `A` (predefined), or `B`/`C` (user-configured)

## Setup

### Using Protocol A (Recommended)
```
AT SP A              -- Select J1939 at 250 kbaud
```

### Using Protocol B at 500 kbaud
```
AT PB 42 01          -- Options=J1939, divisor=1 (500 kbaud)
AT SP B              -- Select protocol B
```

### Using Protocol B at 250 kbaud
```
AT PB 42 02          -- Options=J1939, divisor=2 (250 kbaud)
AT SP B
```

## J1939 CAN ID Structure

### 29-bit ID Breakdown
```
┌─────────┬───────┬───────┬────────────┬────────────┬──────────────┐
│ Priority │  EDP  │  DP   │ PDU Format │ Dest/Group │ Source Addr  │
│ (3 bits) │(1 bit)│(1 bit)│  (8 bits)  │  (8 bits)  │   (8 bits)   │
└─────────┴───────┴───────┴────────────┴────────────┴──────────────┘
```

### PDU1 Format (Destination Addressed)
When PDU Format < 0xF0:
```
Priority [3] | EDP [1] | DP [1] | PDU Format [8] | Dest Address [8] | Source Address [8]
```
PGN = (DP << 16) | (PDU Format << 8) | 0x00

### PDU2 Format (Broadcast)
When PDU Format >= 0xF0:
```
Priority [3] | EDP [1] | DP [1] | PDU Format [8] | Group Extension [8] | Source Address [8]
```
PGN = (DP << 16) | (PDU Format << 8) | Group Extension

### Key Addresses
| Address | Device |
|---------|--------|
| `00` | Engine #1 |
| `03` | Transmission #1 |
| `0B` | Brakes - System Controller |
| `11` | Cruise Control |
| `21` | Body Controller |
| `28` | Cab Controller - Primary |
| `31` | Power Takeoff (PTO) |
| `49` | Instrument Cluster #1 |
| `F9` | Service Tool / Scan Tool (ELM327 default) |
| `FA` | Service Tool (alternate) |
| `FF` | Global Address (broadcast to all) |

## Monitoring PGNs

### Monitor Command
```
AT MP hhhh            -- Monitor PGN 00hhhh (4-digit shorthand)
AT MP hhhhhh          -- Monitor PGN hhhhhh (full 6-digit)
AT MP hhhh n          -- Monitor PGN, stop after n complete messages
AT MP hhhhhh n        -- Monitor PGN (full), stop after n messages
```

**Note**: n counts complete messages, not individual CAN frames. A multiframe response counts as one message.

|-----------|-----------|------|----------------|

| `00FEEE` | 65262 | Engine Coolant Temperature | 1 second |
| `00F004` | 61444 | Engine Speed, Torque | 20 ms |
| `00FEF1` | 65265 | Cruise Control/Vehicle Speed | 100 ms |
| `00FEFC` | 65276 | Dash Display | 1 second |

| `00FEE9` | 65257 | Fuel Consumption | 1 second |
| `00FEEF` | 65263 | Engine Fluid Level/Pressure | 500 ms |
| `00FEF2` | 65266 | Fuel Economy | 100 ms |

### DM1 Active Trouble Codes
```
AT DM1               -- Special command for continuous DM1 monitoring
```

Equivalent to `AT MP FECA` but uses the ELM327's built-in DM1 handler. Press any key to stop.

### Example: Monitor Engine Speed
```
AT SP A              -- J1939 protocol
AT H1                -- Show headers
AT MP F004 2         -- Get 2 engine speed messages
```

Response:
```
6 0F004 00 FF FF F0 7D 00 FF FF
6 0F004 00 FF FF F0 82 00 FF FF
```

## Making J1939 Requests

### Global Request (to all devices)
```
00FEEE               -- Request PGN 00FEEE from all devices
```

Default request ID: `18 EA FF F9` (priority 6, request, to global, from F9)

### Directed Request (to specific device)
```
AT SH EA 00 F9       -- Direct requests to engine #1 (address 00)
00FEEE               -- Request from engine #1
```

### Byte Order
By default (`AT JE` - ELM format), the ELM327 reverses PGN data bytes in requests:
- You send: `00FEEE`
- Wire sends: `EE FE 00` (reversed)

With `AT JS` (SAE format), bytes are sent as-is (you must reverse them yourself).

## Response Formats

### Single Frame Response
```
Headers on:   6 0FEEE 00 8C FF FF FF FF FF FF FF
Headers off:  8C FF FF FF FF FF FF FF
              │                       │
              └── PGN data bytes ─────┘
```

### Multiframe (Transport Protocol) Response
```
Headers on:
012                                          -- Total byte count (hex)
7 0EBF9 00 01 04 FF 50 00 04 0B 54         -- Segment 1
7 0EBF9 00 02 00 00 01 5F 05 02 31         -- Segment 2
7 0EBF9 00 03 6D 05 03 03 FF FF FF         -- Segment 3

Headers off:
012                                          -- Total byte count (hex)
01: 04 FF 50 00 04 0B 54                   -- Segment 1
02: 00 00 01 5F 05 02 31                   -- Segment 2
03: 6D 05 03 03 FF FF FF                   -- Segment 3
```

- `012` hex = 18 bytes total data
- ELM327 handles transport protocol (TP.CM/TP.DT) negotiation automatically
- Max 255 segments x 7 bytes = 1785 bytes per message
- Unused bytes padded with `FF`

### Acknowledgement Response
```
6 0E8FF 00 01 FF FF FF EE FE 00
│ │     │  │              └──── Requested PGN (reversed)
│ │     │  └───────────────── ACK type: 00=positive, 01=negative
│ │     └──────────────────── Source address
│ └────────────────────────── E8=ACK, FF=broadcast
└──────────────────────────── Priority
```

## J1939 Header Display

### AT JHF1 (Default: Formatted)
```
6 0FECA 00 00 FF 00 00 00 00 FF FF
│ │      │
│ │      └── Source address
│ └───────── PGN (with single priority digit)
└─────────── Priority
```

### AT JHF0 (Raw Bytes)
```
18 FE CA 00 00 FF 00 00 00 00 FF FF
│  │  │  │
│  │  │  └── Source address
│  │  └───── PGN low byte
│  └──────── PGN high byte
└─────────── Priority + EDP + DP (5 bits in MSBs)
```

## J1939 Timeout Settings

The ELM327 automatically adjusts timeouts for J1939 (~1.25 seconds default).

```
AT JTM1      -- Timer multiplier 1x (default)
AT JTM5      -- Timer multiplier 5x (for slow/multiframe responses)
```

Use JTM5 when:
- Requesting data that requires multiframe transport
- Similar broadcast data is already flowing (response must wait)
- Getting unexpected `NO DATA` responses

## FMS Standard

Fleet Management Systems (FMS) is a European standard based on J1939:
- Uses only broadcast PGNs (subset of J1939)
- 250 kbaud, gateway-separated
- Monitor with `AT MP <pgn>` commands
- Some data only broadcasts every 10 seconds (VIN, software version)

## J1939 DTC Format

J1939 DTCs are 4 bytes:
```
Byte 1-2: SPN (Suspect Parameter Number) - 19 bits
Byte 3:   FMI (Failure Mode Identifier) - 5 bits
Byte 4:   Occurrence Count + SPN conversion method
```

FMI values:
| FMI | Meaning |
|-----|---------|
| 0 | Data valid but above normal range - most severe |
| 1 | Data valid but below normal range - most severe |
| 2 | Data erratic, intermittent, or incorrect |
| 3 | Voltage above normal or shorted high |
| 4 | Voltage below normal or shorted low |
| 5 | Current below normal or open circuit |
| 6 | Current above normal or grounded circuit |
| 7 | Mechanical system not responding properly |
| 8-10 | Abnormal frequency/pulse/period |
| 11 | Failure mode not identifiable |
| 12 | Bad intelligent device or component |
| 13 | Out of calibration |
| 14-15 | Special instructions |
| 16-18 | Above/below normal - moderate to least severe |
| 19 | Received network data in error |
| 20-30 | Reserved |
| 31 | Condition exists |
